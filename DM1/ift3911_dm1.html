Hypothèses

  1. Un mode de transport doit au moins avoir une section, et que s'il n'y a pas de sections, il n'y a pas d'espaces(siège et cabine)
  2. Un client peut réserver un espace et décider de payer directement ou dans les 24 heures qui suivent sa réservation
  3. Un même client peut faire plusieurs réservations ou paiements mais une réservation/un paiement est fait par un seul client
  4. Pour chaque voyage on a une liste d'escales : infrastructures(aéroport, port, gare) visitées avant d'arriver à déstination
  5. Une compagnie se concentre exclusivement sur un seul domaine d'activité

Contraintes OCL

  1. Un aéroport/gare/port est identifié par trois lettres uniques à chaque aéroport:

    context Infrastructure inv:
    self.nom.size() = 3 and self.nom.isUnique()

  2. La partie alphabétique de l'ID d'un vol est unique à chaque compagnie et la partie numérique est unique à chaque vol au sein de la même compagnie:
    To do

  3. L'aéroport de départ et d'arrivée d'un vol doit être différent:

    Context Vol inv :
    self.pointdeDepart <> self.pointtDarrivee

  4. Tous les sièges/cabines d'une même section ont le même prix:

    Context Section inv:
    self.espace-> forAll(e1, e2 | e1 <>12 e1.prix = e2.prix)

  5. Un itinéraire ne peut pas durer plus de 21 jours:

    context Itineraire inv:
    self.duree <= 504

  6. Le port de départ et d'arrivée doit être le même:

    context Itineraire inv :
    self.pointDepart = pointDarrivee

  7. Un paquebot peut être assigné à plusieurs itinéraires tant qu'ils ne se chevauchent pas :

    Context Paquebot inv:
    self.itineraire ->forAll(i1, i2 | i1 <> i2 implies ( i1.heureDarrivee <= i2.heureDepart )

  8. Toutes les cabines d'une même section ont le prix:

    Context Section inv:
    self.cabine -> forAll(c1, c2 | c1.prix = c2.prix)

  9. Le client peut réserver un siège ou cabine disponible dans un vol (trajet) donné:

    Context Reservation :: reserver()
    Pre : self.espace.estDispo = true
    Post : self.espace.estDispo = false

  10. Un siège réservé devient assigné à un passager une fois payé: le siège est donc confirmé:

    context reservation:: estPaye():Booelan
    Pre: self.espace.estDiso = false and self.espace.estConfirme = false
    Post : self.espace.estConfirme = true

Tâche 5

Justification

  • Un voyage(vol, trajet, itinéraire) contient différentes étapes dont les escales, pour gérer cela nous avons décider
    d'ajouter un attribut "escales" de type arrayList dans la classe “Voyage” qui va gérer les différents escales d'un
    infrastructure(aéroport, gare et port) à un autre

  • la durée d'un itinéraire est convertie en heure car notre variable duree est de type "Heure"

  • Pour qu'une compagnie puisse offrir des voyages il faut que leur mode de transport(avion, train, paquebot) soit
    composé des sections, sans ceci nous n'aurons pas non plus les espaces(siege, cabine); un mode de transport
    qui ne contient pas de siège ne pourrait pas permettre aux clients de voyager.
